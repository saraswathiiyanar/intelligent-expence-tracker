# Use Case Diagram Documentation

## Project Title

**An Intelligent Expense Tracker with Financial Advisory System**

## Actor

**User**

## Use Cases

### 1. Register

**Description:**
The user creates a new account by providing personal details.

**Precondition:**
User is not registered.

**Postcondition:**
A new account is created successfully.

---

### 2. Login

**Description:**
The user logs into the system using email and password.

**Precondition:**
User account exists.

**Postcondition:**
User gains access to the dashboard.

---

### 3. Manage Expenses

**Description:**
The user can add, view, edit, and delete expense records.

**Precondition:**
User is logged in.

**Postcondition:**
Expense data is stored and updated in the database.

---

### 4. View Dashboard

**Description:**
The user views expense summaries and financial information.

**Precondition:**
User is logged in.

**Postcondition:**
Dashboard displays current expense details.

---

### 5. Generate Reports

**Description:**
The system generates expense reports and summaries.

**Precondition:**
Expense records are available.

**Postcondition:**
Reports are displayed to the user.

---

### 6. Get Financial Advice

**Description:**
The system analyzes spending patterns and provides financial recommendations.

**Precondition:**
Expense data exists.

**Postcondition:**
Financial advice is displayed.

---

### 7. View Profile

**Description:**
The user views personal profile information.

**Precondition:**
User is logged in.

**Postcondition:**
Profile details are displayed.

---

## Use Case Relationships

* User → Register
* User → Login
* User → Manage Expenses
* User → View Dashboard
* User → Generate Reports
* User → Get Financial Advice
* User → View Profile

## Conclusion

The system allows users to efficiently track expenses, analyze spending patterns, generate reports, and receive financial recommendations for better money management.
